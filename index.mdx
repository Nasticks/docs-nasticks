## Pourquoi cette documentation ?

Dans une démarche **Docs-as-Code**, je crois fermement que la documentation est aussi importante que le code lui-même. Cet espace me permet de :

1.  **Partager** mes apprentissages avec la communauté.
2.  **Documenter** mes choix pour mes futurs moi (et mes collègues).
3.  **Démontrer** ma capacité à vulgariser des concepts techniques complexes.
